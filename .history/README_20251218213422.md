## Deploy on Fly.io

This repo includes a Dockerfile and `fly.toml` for Fly.io.

1. Install and log in to Flyctl:

```bash
brew install flyctl # macOS
fly auth login
```

2. Create an app (choose a unique name) and set region:

```bash
fly apps create YOUR_APP_NAME
fly regions add iad # or your closest region
```

3. Set required environment secrets (from Supabase project settings):

```bash
fly secrets set \
	NEXT_PUBLIC_SUPABASE_URL=your_supabase_url \
	NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key
```

4. Deploy:

```bash
fly deploy --remote-only
```

5. Open the app:

```bash
fly open
```

Notes:

- The Dockerfile uses Next.js `output: "standalone"` to produce a minimal server.
- `fly.toml` exposes port 3000 internally and enables HTTP health checks.
- Update `primary_region` and `app` name in `fly.toml` as needed.
